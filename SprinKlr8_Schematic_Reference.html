How to Use This Document

Workflow: Open EasyEDA Standard → New Project → New Schematic. Work through each functional block below. For every component, search EasyEDA's library by the LCSC part number (e.g., C84681) — this finds the exact symbol with the correct footprint already attached. Use the Net List section to wire every pin. Use net labels (Place → Net Label) instead of drawing long wires across the sheet — they keep the schematic readable.
Tip: In EasyEDA Standard, press W to place a wire, N to place a net label, P to open the component library search. Searching by LCSC C-number finds the exact part. Use R to rotate components while placing.
Critical — Do not substitute these parts: Use C165255 for the HF46F relay. C128511 is a capacitor and C22363895 is a through-hole relay — EasyEDA search results may show these by mistake. Verify the part description before placing.

5. Auto-Reset Circuit Detail

This circuit allows the Arduino IDE and esptool to automatically reset the ESP32 into bootloader mode before flashing — no manual button pressing required. Replicate exactly as shown.

PinCH340C SignalTransistorESP32 TargetEffect
Q1 RTS# → R6 (10kΩ) → Q1 base
C9 100nF from base to GND		 MMBT3904 NPN
Emitter → GND		 Collector → ESP_EN RTS# low → Q1 conducts → EN pulled low → ESP32 resets
Q2 DTR# → R7 (10kΩ) → Q2 base
C10 100nF from base to GND		 MMBT3904 NPN
Emitter → GND		 Collector → ESP_IO0 DTR# low → Q2 conducts → IO0 pulled low → ESP32 enters bootloader
Sequence: esptool pulses RTS and DTR together in a specific pattern to first assert reset with IO0 low, then release reset — the ESP32 boots into the bootloader and is ready to accept firmware. EN and IO0 have 10kΩ pull-ups to +3.3V (R4 and R3) that return them high when neither button nor transistor is pulling them low.

6. Relay + LED Circuit Detail

The zone LED and relay coil share the same driver net. When the ULN2803A output is pulled low (zone active), current flows through both the relay coil and the LED simultaneously.

7. EasyEDA Standard Entry Tips

Recommended Entry Order

Work through the schematic in this order to make routing logical:

  1. Power symbols first — place GND, +5V, +3V3, VBUS_RAW power flags on the sheet
  2. Power supply block — D1, U4, L1, D2, C1, C2, F1, U3, C3, C4
  3. ESP32 module (U1) — place centrally, add decoupling C5, C6
  4. CH340C (U2) + auto-reset — Q1, Q2, R6, R7, C9, C10
  5. USB-C connector (J1) + ESD (D3) + F2 + R1, R2
  6. ULN2803A (U5) + 8 zone GPIO connections using net labels
  7. Relays K1–K8 + zone LEDs LED1–LED8 + R8–R15
  8. Zone output terminals J4–J11, J12 COM, MOVs RV1–RV8
  9. Power input connectors J2, J3, JP1, F1
  10. Buttons SW1, SW2, pull-up/pull-down resistors R3, R4, R5
  11. Status LED9 + R16

Library Search in EasyEDA Standard

Fastest method: Press P → in the search box type the LCSC part number exactly (e.g., C84681) → click the result → the component appears with footprint pre-assigned. Do this for every IC and connector.
For basic passives (resistors, capacitors): search 0402 resistor 10k or similar. EasyEDA's basic library has these — they count as "Basic" parts for JLCPCB with no setup fee.

Net Labels vs. Wires

Use net labels (press N) for any connection longer than ~3 component widths. Give each label the exact net name from this document (e.g., RELAY_DRV_1). EasyEDA treats any two labels with the same name as electrically connected — no wire needed across the sheet.

Power Flags

EasyEDA expects at least one "Power Flag" on each power rail to suppress ERC warnings. Place a PWR_FLAG symbol on +5V, +3V3, GND, and VBUS_RAW nets.

ERC Check

When the schematic is complete: Design → ERC Check. Common errors to expect and resolve:

  • "Pin unconnected" — verify every IC pin is either connected to a net or has a "No Connect" (X) marker
  • "Net has no driver" — add a PWR_FLAG on rails that only have passive components
  • "Duplicate net labels" — intentional (they're connected); dismiss these

Before Generating Gerbers

Double-check before PCB layout: ✓ U5 COM pin (pin 10) connected to +5V  |  ✓ R5 from GPIO12 to GND present  |  ✓ R1, R2 (5.1kΩ) on USB-C CC1 and CC2  |  ✓ K1–K8 using LCSC C165255 (not C128511 or C22363895)  |  ✓ D1 MB10F AC pins to AC_LINE and AC_COM (not polarity-sensitive, but verify symbol orientation)
